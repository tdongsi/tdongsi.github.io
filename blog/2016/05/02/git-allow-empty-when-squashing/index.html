Git: Allow-empty When Squashing

May 2nd, 2016 12:15 am

http://git.661346.n2.nabble.com/Confusing-error-message-in-rebase-when-commit-becomes-empty-td7612948.html

that message comes from “commit –amend”, which is called by rebase to handle the squash. The “repeat your command” part is confusing. The right thing to do here is:

git commit –amend –allow-empty

if you want to have an empty commit, or:

git reset HEAD^

if you want to have nothing.

Of course the first one would never occur to you, because it is not “your command” in the first place. :)

We could change it to say “use git commit –amend –allow-empty”
